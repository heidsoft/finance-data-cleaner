# 数据处理 Tab 体验优化设计方案

## 背景

当前「数据处理」Tab 已有基础清洗和合并功能，但操作无确认、无预览、反馈不清晰，财务人员使用有风险感。目标是让财务导入 → 清洗 → 合并 → 导出在一个地方顺畅完成。

## 功能列表

### 1. 操作前确认对话框
每个清洗/去重操作执行前弹出确认框，显示影响范围，用户确认后才生效。

**涉及操作：**
- 去重（显示将删除 N 行重复数据）
- 清空（显示将删除 N 行空行 + M 列空列）
- 日期规范化（显示将转换 N 个日期格式）
- Trim（显示将处理 N 行中的空格）
- 填空值（显示将填充 M 个空单元格）
- 选列（显示将保留/删除哪些列）

**UI 形式：** 轻量级模态对话框，固定在屏幕中央，显示操作摘要 + 取消/确认按钮。

---

### 2. 合并预览（新增）
多个文件合并前，先显示预览界面：
- 显示合并后总行数、总列数
- 列名对比表：每个文件包含哪些列，哪些列是新增的、哪些列缺失
- 自动列名对齐：相同含义的列名（如"订单金额"和"销售额"）智能匹配，用颜色标注

**UI 形式：** 合并按钮点击后弹出预览面板，用户确认后才执行合并。

---

### 3. 导出格式选择（新增）
点击导出按钮时，弹出格式选择面板：
- 文件格式：CSV / Excel
- 编码选择（仅 CSV）：UTF-8 / GBK
- 分隔符选择（仅 CSV）：逗号 / 制表符

**UI 形式：** 导出按钮组改为下拉菜单或弹出面板，而非直接导出。

---

### 4. 操作结果提示（新增）
每个操作完成后，在界面底部显示操作结果 toast 提示，3 秒后自动消失：
- ✅ 去重完成：删除了 3 行重复数据
- ✅ 清空完成：删除了 2 行空行、1 列空列
- ⚠️ 日期规范化：发现 5 个无法解析的日期格式，已跳过

---

### 5. 一键清洗按钮（优化）
在现有分散的清洗按钮基础上，增加一个「一键清洗」按钮，将以下操作一次完成：
1. Trim 去除前后空格
2. 清空空行空列
3. 日期规范化

点击前显示确认："将执行：去空格、清空空行空列、日期格式标准化，是否继续？"

---

## 组件设计

### ConfirmDialog 组件
```
Props: { title, message, onConfirm, onCancel, confirmLabel, cancelLabel }
- 固定底部居中，白色卡片，阴影遮罩
- 标题 + 描述文字 + 双按钮
- ESC 键或遮罩点击 = 取消
```

### MergePreview 组件
```
Props: { files, onConfirm, onCancel }
- 显示文件列表和列名对比
- 高亮缺失列/新增列
- 合并后行数/列数统计
- 确认/取消按钮
```

### ExportPanel 组件
```
Props: { onExport, onCancel }
- 格式单选 + 编码/分隔符下拉（条件显示）
- 确认导出按钮
```

### Toast 组件（全局）
```
Props: { message, type: 'success' | 'warning' | 'error' }
- 固定在状态栏上方
- 自动 3 秒消失
- 可手动关闭
```

## 实现顺序

1. Toast 组件（最简单，全局可用）
2. ConfirmDialog 组件 + 集成到各个操作
3. 操作结果 toast 反馈
4. 导出格式选择面板
5. MergePreview 组件 + 合并预览
6. 一键清洗按钮
