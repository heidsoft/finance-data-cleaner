# 退款损失精确计算 - 设计规范

## 1. 概述与目标

**问题现状**：当前退款损失使用平台平均佣金率估算（默认5%），每笔订单佣金相同，无法反映真实佣金结构。

**改进目标**：支持导入订单级佣金明细，按「订单号+平台」精确匹配实际佣金，无法匹配时降级为估算并标注。

---

## 2. 数据模型

### 2.1 佣金明细 (CommissionDetail)

| 字段 | 类型 | 说明 |
|------|------|------|
| orderId | string | 订单号 |
| platform | string | 平台名称 |
| commission | number | 佣金金额（元） |

**来源**：平台导出的佣金明细文件（CSV/Excel）

### 2.2 退款损失记录 (RefundLossRecord)

| 字段 | 类型 | 说明 |
|------|------|------|
| orderId | string | 订单号 |
| refundAmount | number | 退款金额 |
| platform | string | 平台 |
| commission | number | 匹配到的实际佣金 |
| isMatched | boolean | true=精确匹配，false=估算 |
| lossAmount | number | 损失金额（等于commission） |

---

## 3. 功能设计

### 3.1 新增「导入佣金明细」按钮

**位置**：账单对账 Tab → 佣金自动计提表区域

**交互**：
1. 点击按钮 → 打开文件选择框（仅 CSV/XLSX/XLS）
2. 选择文件 → 自动解析，提取 orderId、platform、commission 字段
3. 解析完成 → 显示佣金明细预览（行数、平台分布）
4. 数据存储到 state：`commissionDetails[]`

**字段匹配规则**：
- orderId：优先匹配含"订单号"、"order"、"编号"、"id"的列
- platform：优先匹配含"平台"、"platform"、"渠道"的列
- commission：优先匹配含"佣金"、"commission"、"服务费"、"扣点"的列
- 若关键列找不到，显示警告并允许手动选择列

### 3.2 佣金明细查看

**位置**：导入按钮旁边

**功能**：
- 显示已导入佣金明细行数
- 点击展开查看明细（分页 Table）
- 支持按平台筛选

### 3.3 修改退款损失计算逻辑

**原逻辑**：
```
avgRate = totalCommission / totalAmount
loss = sum(refundAmount) * avgRate
```

**新逻辑**：
```
FOR each refundOrder:
  1. 在 commissionDetails 中查找 orderId + platform 匹配
  2. IF 找到:
       loss = matched.commission
       isMatched = true
     ELSE:
       loss = refundAmount * avgRate  // 降级估算
       isMatched = false
       标记 ⚠️

lossAmount = sum(loss)
```

**平均佣金率来源**：
- 优先使用已导入账单记录的平均佣金率
- 若无账单记录，使用 5% 默认值

### 3.4 退款损失表输出增强

**新增列**：

| 列名 | 说明 |
|------|------|
| 匹配状态 | ✅已匹配 / ⚠️估算 |
| 实际佣金 | 匹配到的真实佣金金额 |
| 佣金来源 | "精确匹配" / "均摊估算" |

**尾部汇总**：
- 退款总笔数
- 已匹配笔数 + 金额
- 估算笔数 + 金额
- 预估佣金损失总额

### 3.5 匹配率提示

**展示位置**：退款损失表顶部

**格式**：
```
已匹配 156/180 笔 (86.7%)
⚠️ 24笔无法精确匹配，使用估算
```

## 4. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 佣金文件格式错误 | Toast提示"文件格式不支持"，终止导入 |
| 佣金文件无有效数据 | Toast提示"未找到佣金数据"，终止导入 |
| 订单号在退款中不存在 | 静默跳过，不影响其他匹配 |
| 平台字段不匹配 | 视为不同订单，不匹配 |

---

## 7. 降级策略

当以下情况发生时，自动降级到估算模式：

1. 未导入佣金明细文件
2. 佣金文件中找不到对应订单号+平台
3. 佣金字段解析失败

降级后显示 ⚠️ 警告，用户可选择：
- 继续使用估算结果
- 补充导入佣金明细后重新计算

---

## 8. 测试用例

| 场景 | 预期结果 |
|------|----------|
| 退款订单全部匹配 | 100% ✅，无估算 |
| 退款订单部分匹配 | 显示匹配率，估算部分标⚠️ |
| 未导入佣金明细 | Toast提示，显示估算模式 |
| 订单号匹配但平台不匹配 | 视为不匹配，走估算 |
| 佣金金额为0 | 正常计入，损失为0 |

---

## 9. 非功能性要求

- 佣金明细文件支持最大 50万行
- 匹配算法时间复杂度 O(n*m)，需优化为 Map 查找 O(n)
- 导入过程显示进度条
